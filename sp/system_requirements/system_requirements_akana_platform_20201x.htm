System Requirements for Akana Platform 2020.1.x

The following table provides information about the minimum system requirements for running Akana Platform/Akana API Platform 2020.1.x, and about associated supported technologies, such as API description document formats.

For instructions for installing or upgrading, see Installing the Akana API Platform 2020.1.x or Upgrading Akana API Platform to Version 2020.1.x.

Supported Platforms: 2020.1.x

Requirements

  • Akana Platform Host
  • Client browser for accessing developer portal
  • Database versions
  • Elasticsearch (required for Community Manager only)

Supported Technologies

  • API description document formats
  • Third-party identity and access management (IAM) technologies

Requirements

Name Requirement
Akana Platform Host

Hardware per Akana container:

2Ghz CPU, dual core, 4GB RAM minimum, 6GB recommended.

Operating Systems:

  • Windows—Windows Server 2012 R2, 2016
  • Linux—CentOS Linux 7.x or equivalent

Memory configuration:

Default maximum heap size for all Akana Containers (Java processes): 2048 MB.

Recommended: 4096 MB

Note: The above assumes a dedicated CPU for each container.

JRE version:

  • The Akana Platform is compatible with JRE 1.8 and is shipped with OpenJDK JRE version 1.8.0_212. You can configure the platform to use an external JRE, of a compatible version, if preferred.
Client browser for accessing developer portal

All themes:

  • Firefox version 72.0 and later
  • Chrome version 70 and later
  • Internet Explorer versions 11.1282 and later

    Note: for information about running the Akana Administration Console in Internet Explorer 11, see Troubleshooting: IE browser support.

  • Microsoft Edge version 44.18362 and later.
Database versions

Akana API Platform 2020.1.x supports the following database versions:

  • MySQL 8.0

    Database driver required: see Database drivers (install doc).

  • Oracle 12c Release 2, Oracle 18c (18.3), Oracle 19c.

    Note: The Oracle database password must not start with a number or special character.

    Database driver required: see Database drivers (install doc).

  • Microsoft SQL Server 2012 SP4, 2016 SP2
  • MongoDB (Windows 64-bit MSI and Linux 64-bit TGZ): 3.6 versions up to 3.6.16.

    To download a MongoDB file for your platform, refer to the MongoDB Downloads website.

Note: Depending on the database type and version that you're using, there might be specific requirements, constraints, or additional steps needed. For details, refer to the Database Notes section of the installation documentation.

Elasticsearch

(Required for Community Manager only)

Supported versions:

  • 6.8.6 and later 6.x.x versions

Recommended for the standalone Elasticsearch server: 4GB RAM, 25GB disk space

Note: Depending on the usage and the number of objects, more disk space might be needed.

See Installing and Configuring Elasticsearch.

Supported Technologies

Name Details
API description document formats

The platform supports adding an API by uploading, or otherwise referencing, API description documents in any of the following formats:

  • For REST APIs:
    • OAS 3.0
    • Swagger 2.0
    • Swagger 1.2
    • Swagger-YAML
    • RAML 0.8
    • WADL
  • For SOAP APIs:
    • WSDL

For more information, see API Description Language Support (Developer Portal documentation).

Third-party identity and access management (IAM) technologies

OAuth

  • 1.0a, 2.0

OpenID Connect

  • 1.0

Kerberos

  • Supported on Windows Server versions 2012, 2012 R2, and 2016

SAML Identity System

  • 1.1, 2.0

SAML Web SSO

  • The Akana API Platform solution for using SAML for single sign-on supports SAML Version 2.0. For more information, see SAML Web Browser SSO Support in the Akana API Platform. Supported where the provider supports the implementation of SSO with SAML 2.0.

SiteMinder

  • R12.0 SP3

PingFederate:

  • 9.3.x, 10.0.x
