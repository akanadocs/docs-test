System Requirements for Akana Platform 2020.1.x

The following table provides information about the minimum system requirements for running Akana Platform/Akana API Platform 2020.1.x, and about associated supported technologies, such as identity systems and API description document formats.

For instructions for installing or upgrading, see Installing the Akana API Platform 2020.1.x or Upgrading Akana API Platform to Version 2020.1.x. {install and upgrade doc links to be updated, including links below}

Akana Platform Host

Hardware:

Single CPU, 2Ghz, 2GB RAM (minimum)

Operating Systems:

  • Windows—Windows Server 2012 R2, 2016
  • Linux—CentOS Linux 7.x or equivalent
Client browser for accessing developer portal

All themes:

  • Firefox version 72.0 and later
  • Chrome version 70 and later
  • Internet Explorer versions 11.1282 and later

    Note: for information about running the Akana Administration Console in Internet Explorer 11, see Troubleshooting: IE browser support.

  • Microsoft Edge version 44.18362 and later {MM: do we still need this note?}
Database versions

Akana API Platform 2020.1.x supports the following database versions:

  • MySQL 8.0

    Database driver required: see Database drivers (install doc).

  • Oracle 12c Release 2, Oracle 18c (18.3), Oracle 19c.

    Note: The Oracle database password must not start with a number or special character.

    Database driver required: see Database drivers (install doc).

  • Microsoft SQL Server 2012 SP4, 2016 SP2
  • MongoDB (Windows 64-bit MSI and Linux 64-bit TGZ): 3.6 versions up to 3.6.16.

    To download a MongoDB file for your platform, refer to the MongoDB Downloads website.

Note: Depending on the database type and version that you're using, there might be specific requirements, constraints, or additional steps needed. For details, refer to the Database Notes section of the installation documentation.
Memory configuration

Default maximum heap size for all Akana Containers (Java processes): 2048 MB.

Recommended: 4096 MB
Elasticsearch

Supported versions:

  • 6.8.6 and later 6.x.x versions

Recommended for the standalone Elasticsearch server: 4GB RAM, 25GB disk space

Note: Depending on the usage and the number of objects, more disk space might be needed.

See Installing and Configuring Elasticsearch.
API description document formats supported

The platform supports adding an API by uploading, or otherwise referencing, API description documents in any of the following formats:

  • For REST APIs:
    • OAS 3.0
    • Swagger 2.0
    • Swagger 1.2
    • Swagger-YAML {someone please let me know if this is not correct. It's in the doc}
    • RAML 0.8
    • WADL {We say we don't support WADL for API doc. But the doc, and the UI (Add API page), both say we support it.}
  • For SOAP APIs:
    • WSDL {is version a factor here?}

For more information, see API Description Language Support (Developer Portal documentation).
