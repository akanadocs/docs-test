System Requirements for Akana Platform 2018.0.0

The following table lists the minimum system requirements for running Akana Platform 2018.0.0

For instructions for installing or upgrading, see Installing the Akana API Platform 2018.0.0 or Upgrading Akana API Platform from Version 8.4x to Version 2018.0.x.

Supported Platforms: 2018.0.0

Component Name Requirement
Akana Platform Host

Hardware:

Single CPU, 2Ghz, 2GB RAM

Operating Systems:

  • Windows—Windows Server 2008, Windows Server 2012, Windows 10 {Per DR question on 2008}
  • Linux—Red Hat Enterprise Linux 5.8, 6.2, 7
Client browser for accessing developer portal

Default Theme and Hermosa Theme:

  • Firefox version 25 and later {current version is 56}
  • Chrome version 31 and later (current version is 68.0}
  • Internet Explorer versions 9, 10, and 11 {MS doesn't support 9 and 10 since Jan 2016}

Simple Developer Theme:

  • Firefox version 32 and later {current version is 56}
  • Chrome version 38 and later (current version is 68.0}
  • Internet Explorer versions 9, 10, and 11 {MS doesn't support 9 and 10 since Jan 2016}
Database versions

Akana API Platform 2018.0.0 supports the following database versions:

  • MySQL 5.6 and 5.7 {Per DR 5.7 is new for 2018.0.0}

    Note: If you're using MySQL 5.6 with version 8.0.x or later of Policy Manager, the Akana Platform, or Community Manager, you must configure some specific MySQL property settings. See MySQL Database Property Settings.

  • Oracle 11g, 12c, and 12c Release 2
  • Microsoft SQL Server 2012, 2014, and 2016. {Per DR poss deprecate 2012}

    Note: If you're using Microsoft SQL Server, you must set the quartz trigger property in the Akana Administration Console to True. See Database notes: Microsoft SQL Server.

The following database versions will be certified at a later date:

  • DB2 9.7 and 10.5 {Per DR 9/17 DB2 10.5 definitely will be certified.} {for DB2 notes the schema wizard will not work and they have to create the database and user manually}

Note: Depending on the database type and version that you're using, there might be specific requirements, constraints, or additional steps needed. For details, refer to the Database Notes section of the installation documentation.
Memory configuration

The default maximum heap size for all Akana Containers (i.e., Java processes) is 2048 MB.

Recommended: 4096 MB. {MM or GW is this enough?}
MongoDB

Supported versions (Windows 64-bit MSI and Linux 64-bit TGZ):

  • MongoDB versions 3.2.12, 3.2.13, and later versions up to 3.2.18 {Per DR new is 3.6.x not sure which version. Not sure about 3.4}

To download a MongoDB Version 3.2.18 file for your platform, refer to the MongoDB Downloads website.
Elasticsearch Supported versions: 6.3.1
