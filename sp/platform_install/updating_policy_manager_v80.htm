Introduction

Akana periodically issues updates for Policy Manager.

  • Update files can be downloaded from the Akana Customer Support Site.
  • Update files must be copied to the Akana Platform Release Directory.
  • Each update file must be extracted in version number order (earliest version first).
  • Updates must be applied to all container instances.

Note: For new installation, the update process is included as part of the Installing Policy Manager 8.0 topic.

Confirm Installed Updates

If you need to confirm which updates (Akana Platform or Policy Manager) are installed perform the following steps:

  • Launch the Akana Administration Console and select the Installed Features tab.
  • To view bundles associated with the current update, select the Installed Features tab, select Bundle from the Filter drop-down, and click the Version column to sort by version.
  • To view Bundle Details click on a bundle line item.

Manually Installing Schemas

If you have a requirement to manually install the Akana schemas, contact SOA Software Customer Support prior to beginning this installation to obtain a series of schema installation scripts and additional instructions.

Update Existing Policy Manager Installation (Manual Installation)
  1. Launch www.support.akana.com.
  2. Go to Downloads > PolicyManager > 8.0 > Updates > <Update Version>.
  3. Download the update ZIP files.
  4. Extract the platform ZIP files to the Akana Platform Release Directory in version number order.
  5. When the "Confirm file replace" dialog displays, click Yes to All. The automated ZIP file then updates a series of files in the release directory folders.
  6. Stop the container instance the updates will be applied to. See Starting and Stopping a Container Instance for start/stop options.
  7. After applying the updates, delete the /instances/configurator/cache directory if it exists.
  8. Launch the Configure Container Instance Wizard. Navigate to the Platform release directory c:\<release directory>\bin and enter:

    - startup.bat configurator (Windows)
    - startup.sh configurator (UNIX)
  9. On the Instance Name screen, specify the name of the container instance the updates will be applied to.
    • If you need to verify the container instance name, you can find them in the /instances folder.
    • Note that a container instance name is case sensitive.
  10. On the Instance Already Exists screen, click Update to update the current container instance. Click Next, verify that the container is stopped, then click Finish to apply the updates.
  11. After the update is complete a dialog box displays showing the number of bundles that have been updated.
  12. Start the container instance the updates were applied to. See Starting and Stopping a Container Instance for start/stop options.
  13. Perform the following prerequisite steps:
    • Deploy Database Driver: - Before performing the database configuration in he Akana Administration Console, verify that a database driver for the database used with the current container configuration is deployed to the<Release Directory>\instances\<container instance>\deploy folder. If a database driver is not deployed, copy the database driver to the \deploy directory. Refer to
    • Clear Browswer Cache: Before launching the Akana Administration console, clear the browser cache. This is necessary to ensure that user interface changes included in the Akana Platform updates display properly.
    • Manually Installing Policy Manager Schemas: If you have a requirement to manually install the Policy Manager schemas, contact Akana Customer Support prior to beginning this installation to obtain a series of schema installation scripts and additional instructions.
  14. Users that will not be utilizing the Akana Administration Console can skip the remainder of this procedure
  15. After successfully starting the container instance, deploying the database driver, and clearing the browser cache, launch the Akana Administration Console for the updated container instance:

    Enter: http://<hostname>:<port>/admin
  16. Select the Admin Console domain, enter the Username and Password, and click Login. The Akana Administration Console launches and displays the Available Features tab.
  17. If you have previously installed the Policy Manager Services feature, verify if additional schemas must be installed using the Manage Schemas Wizard.
  18. To perform this task, in the Akana Administration Console, click the Configure tab. In the Configuration Actions section click Manage Schemas. The Manage Schemas Wizard launches and displays the Install Schemas screen. In the Available Schemas section, select the checkbox of the available Policy Manager schema and click Finish.

    Note:     If you have not previously installed the Policy Manager Services feature, skip this section and refer to Using the Policy Manager Service Feature. Complete the procedure. During the database and schemas configuration process select all Available schemas on the Manage Schemas Wizard screen.
  19. After the configuration tasks are complete, navigate to the Repository tab and verify that the repository for the installed update is present. If it is not, click the Refresh icon to update the repository.
  20. As a final step, navigate to the "System" tab and click Restart.
  21. The update process is now complete.
