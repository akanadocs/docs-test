Introduction

Envision is a data visualization tool that integrates with the Policy Manager Management Console.

Envision Installation Selections

Envision installation options include one feature and three plug-ins.

Envision Feature

The Envision feature installs the standalone Analytics Manager user interface and associated services.

This feature must be installed into the Policy Manager container instance. Requires that the Policy Manager Console feature be installed to the Policy Manager container instance.

Note: Make sure that MongoDB is set up and started before proceeding with this installation, so that you can:

  • Connect to the MongoClientURI.
  • Modify the associated policies after logging into the Policy Manager Management Console.

As part of installing this feature, when you click Configure, in the Configure MongoDB Database Wizard, you'll need to provide the MongoClientURI. See Configure MongoDB Database Wizard (Envision only).

Envision Plug-Ins

The following plug-ins are applicable to Envision:

  • Akana Envision Metrics Collector
  • Akana Envision Policy Manager Console Extensions
  • Akana Envision Service Extensions
  • Akana Policy Manager Security Provider for Envision
  • Akana Sample Datasets for Demo Charts

Akana Envision Metrics Collector

The Akana Envision Metrics Collector plug-in is technically an optional plug-in, but is needed in order to retrieve the metrics data from the API Gateway and channel it to Envision. Without this plug-in you could pull data from another source, but not from the API Gateway (Network Director).

Installs extensions that will capture metrics from Network Director for use by Envision.

This plugin is installed on a container with the Network Director feature. It adds the handler for the Business Metrics policy and the supporting extraction and reporting capabilities.

Requires the Network Director or Policy Manager Services feature.

If Network Director and Envision are installed in separate containers, install the Akana Envision Metrics Collector in the container where Network Director is installed.

Configuration:

No configuration required.

Akana Envision Policy Manager Console Extensions

The Envision Console Extensions plug-in installs extensions to the Policy Manager Management Console to direct the collection of business metrics and SLA information. The information is then fed into the Envision product. The extensions include configuration screens for the policies relating to business metrics and SLA.

This plugin is installed on a container with the Akana Policy Manager Console feature. It adds the user interface for the Business Metrics policy and the Business QoS (SLA) Policy.

Configuration:

This plugin has only one configuration setting which is the same as the Akana MongoDB Support plugin: see Akana MongoDB Support Plugin.

Akana Envision Service Extensions

This plug-in installs connectors that Policy Manager uses for collection of business metrics and SLA collection that are then fed into the Envision product. This feature must be installed into the Policy Manager container instance and works in conjunction with Akana Envision. This plug-in also automatically installs the Envision Metrics Collector plug-in.

This plugin is installed on a container with the Akana Policy Manager Services (Management and Security) and/or Policy Manager Jobs features. It adds the metrics API replacement that supports business metrics and storage in the MongoDB NoSQL database as well as the SLA Engine extension for Business QoS Policies.

Installs the following policies:

  • Business Metrics Policy to the Policies > Operational section.
  • Business Service Level Policy to the Policies > QoS section.

This feature requires the Policy Manager Console feature.

Configuration:

This plugin has only one configuration setting which is the same as the Akana MongoDB Support plugin: see Akana MongoDB Support Plugin.

Akana Policy Manager Security Provider for Envision

This plugin is installed on a container with the Akana Policy Manager Services and Envision feature. It installs the security plugin for protecting API Gateway collected operational metrics.

This plugin has no configuration.

Akana Sample Datasets for Demo Charts

This optional plugin installs sample datasets for use in the Envision Console. See Using the Akana Sample Datasets for Demo Charts Feature.

Installation and Configuration

To install the Envision plug-ins:

  1. Log into the Akana Administration Console of the target installation container instance using your default credentials.
  2. On the Available Features screen, select Plug-ins from the Filter menu.
  3. Select the Envision Console Extension plug-in and click Install Feature.
  4. After the installation is complete, click Configure and continue.
  5. On the Configure MongoDB Database screen, specify the host name and port where the MongoDB database is installed in the MongoClientURI field. The default MongoDB port is 27017.
  6. Click Finish, and then click OK to restart the container. The URI is assigned and the system automatically configures seed data and adds schemas.

    After the installation is complete, you can manage datasets(install / uninstall) via the Configuration tab by selecting Configure MongoDB Database.
