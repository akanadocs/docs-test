com.soa.console.policy.wsa.help.anonymous
This option allows "Anonymous" Endpoints to initiate Message Exchanges and receive Replies. It is used by Endpoints that cannot have a stable, resolvable URI. Therefore a Response cannot be sent separately to a Response Endpoint and must be returned synchronously in the Body of the Response to the Input Message.
com.soa.console.policy.wsa.help.detail
To configure the SOAP Response Endpoint type, click the checkbox for "Support Anonymous Messages" or "Support Non-Anonymous Messages" then click the radio button of the WS-Addressing specifications option in the "Supported Versions" section. You can select "All" supported WS-Addressing specifications, or select one or more versions from the "Supported Versions" list box. After completing your selections, click "Finish" to complete the configuration session.
com.soa.console.policy.wsa.help.nonanonymous
This option allows "Non-Anonymous" Endpoints to initiate Message Exchanges and receive Replies. It is used by Endpoints that have strict URI requirements based on selected WS-Addressing specification(s). Response Messages for IN-OUT exchanges will be sent to the specified Endpoint. There will be no Body Content in the synchronous Response to the Input Message.
com.soa.console.policy.wsa.help.paragraph
The "Specify WS-Addressing Policy Options" screen allows you to configure the type of SOAP Response Endpoints required for returning responses to Input Messages. "Anonymous" and "Non-Anonymous" options are supported. The "Supported Versions" section allows you to select the WS-Addressing specifications that are supported for the current WS-Addressing policy.
