com.soa.console.policy.opscript.dyn.page1.description
<p>The <strong>Modify Script Policy</strong> screen allows you to update a policy using a defined script language. The script can be written in any of the languages available in the drop-down list.</p><p>You can import an existing script by importing it to the current script definition via the <strong>Imports</strong> section. To accomplish this, in the <strong>Available Scripts</strong> section select a script from the tree, then use the left arrow to copy it to the <strong>Imported Scripts</strong> section. After the script is imported it can then be referenced in the script source (e.g., samplescript(value1,value2);).</p><p>Note that only scripts that match the language type will be available for selection in the tree. The script does not return a value.</p><p>Several predefined functions and variables are available to build a policy expression that is evaluated at runtime.</p><p>From the script workspace, to use a pre-defined method, type CTRL+Space, choose from the list, and then type a period (.) after the method.</p><p>When you are ready to save your script, click <strong>Finish</strong>.</p>
