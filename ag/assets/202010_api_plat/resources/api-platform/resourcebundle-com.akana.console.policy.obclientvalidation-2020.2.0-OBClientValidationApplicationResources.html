com.akana.console.policy.obclient.help.paragraph
<p>The Client Certificate can be specified using HTTP headers containing the client''s certificate and the Subject DN and Serial Number extracted from the certificate. These headers are commonly set by the network device that is terminating the SSL session (for example, load balancer).<p>The platform can identify the client: a) From the Consumer Subject Category. This requires an authentication policy attached to the API, such as OAuth Security Policy. b) From the client_id in the request parameters (query or form).<p>The certificate is validated against the client''s JWKS URL configured with keys having "use":"tls".</p>
