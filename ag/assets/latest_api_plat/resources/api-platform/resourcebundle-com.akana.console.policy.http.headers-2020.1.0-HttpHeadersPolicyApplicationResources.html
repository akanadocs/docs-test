com.akana.console.policy.http.headers.help.paragraph
<p>On this page you can configure the high-level settings that determine how the HTTP Headers Injection Policy will work.</p><p><strong>Protection Scope</strong>: Identifies which messages in a message exchange are governed by the policy. Choose any combination of <strong>OUT</strong> or <strong>FAULT</strong>.</p><p><strong>Role</strong>: Indicates the policy is applied to messages from the client (<strong>Provider</strong>).</p><p><strong>Headers</strong>: You can specify one or more HTTP headers to be injected; check the box and specify the header name and value. A few security header examples are: <ul><li>X-XSS-Protection: 1; mode=block</li><li>Strict-Transport-Security: max-age=31536000; includeSubDomains</li><li>X-Content-Type-Options: nosniff</li><li>Cache-control: no-store, no-cache</li><li>Content-Security-Policy: default-src &apos;self&apos;</li><li>X-Frame-Options: SAMEORIGIN</li></ul></p>
