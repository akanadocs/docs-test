Introduction

The "Insert XML Activity" is a DataPower Orchestration Policy Activity that allows you to customize the message that will go to the back-end service. For example, you can take the result of a previously invoked service activity, take an XML snippet or element and append it to the back-end service call of another invoked service activity. This makes a call to another web service, obtains information, and then uses it on the final back-end service call. The configuration is performed by assigning a unique name to the activity and defining message type and XPath for From and To variables.

Configuration Options

The policy includes the following configuration options:

Name

  • Allows you to specify a unique name for the activity.
  • The name should describe what function the activity is performing.

From Variable

This section is used to specify where you want to get the XML element from.

  • Message - Select if you would like to get the XML element from the default message.
  • User-Defined - Select if you would like to get the XML element  from a user defined message. Specify a user defined message in the text box.
  • XPath Query - Specify the XPath that represents the element you want to take from that message.

To Variable

This section is used to specify where you want to attach the XML element to.

  • Message - Select if you would like to attach the XML element to the default message.
  • User-Defined - Select if you would like to attach the XML element  to a user defined message. Specify user defined message in the text box.
  • XPath Query - Specify the XPath that represents where you want to attach or append the XML element that is matched in the second XPath message.
