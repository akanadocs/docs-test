About the WS-Security Verify Signature Component

Policy Manager provides a "WS-Security Verify Signature Component" that is used to verify signatures based on the OASIS WS-Security Standard.

Note: The WS-Security Verify Signature implementation complies with the "Signatures > Signature Validation" specification outlined in the "WS-Security SOAP Message Security 1.0" specification.

For general information on adding this policy component to the Request or Response Configuration of your policy and available configuration options see Configure Policy Components.

Creating a WS-Security Verify Signature Component policy

The first step in creating a policy is to define the basic policy information. Then, you can configure the policy details.

To add an operational policy
  1. Go to Workbench > Browse > Organization, and select Policies > Operational Policies. The Policies Summary is displayed.
  2. Click Add Policy.
  3. Choose the policy type and click Next.
  4. Specify a name (required) and description (optional) and click Finish. The Add Policy Wizard creates a draft policy instance that you can then configure on the Policy Details page.

For more information, see Add Policy.

Enter the following "WS-Security Verify Timestamp Component" configuration information:

Verify Timestamps

Select the timestamp verification method you would like to apply to the message:

Verify Expiration Timestamp Only

A radio button that enables the verification of an Expiration Timestamp only. This is the default selection.

Verify Expiration Timestamp or Creation Timestamp

A radio button that verifies the Expiration Timestamp and Creation Timestamp. To configure the Creation Timestamp enter the number of seconds the represents the expiration time of the Creation Timestamp. This is a required field.

Advanced Options

Actor

Enter the URI of the actor attribute of the WS-Security Header that includes the timestamp header you would like to verify. The actor attribute can be used to indicate which recipient should process the verification of the timestamp header. If the actor is omitted, it means that verification of the timestamp header is targeted to final endpoint. Use of this feature is optional.

Click the Audit checkbox if you would like to log all activity associated with the execution of the "WS-Security Verify Timestamp Component." If this option is not selected, the system logs failure only. Use of this feature is optional.

WS-Security Verify Timestamp Component Key Functionality

The "WS-Security Verify Timestamp Component" is a Policy Manager policy component that can be configured in a Request or Response Configuration of a policy definition. Each component should be placed in a Request or Response Configuration relative to how message requests and responses will be processed. The following section provides an overview of key functionality.

Component Functionality

Component Placement in Policy Configuration

The "WS-Security Verify Timestamp Component" can be placed in the Request or Response Configuration.

Verify Timestamps

This component provides two timestamp verification options:

Verify Expiration Timestamp only

Verifies the Expiration Timestamp expiration as its default behavior. At runtime, a SOAP message will be rejected if its expiration time is before the current time.

Verify Expiration Timestamp and Creation Timestamp

Verifies both the Expiration Timestamp and Creation Timestamp. The Creation Timestamp expiration is configured by specifying a message expiration time in seconds. At runtime, this component will check the following two conditions:

If the expiration Timestamp of a SOAP message is before the current time, the message will be considered invalid and be rejected.

If the creation Timestamp of a SOAP message is before the current time minus the message expiration time, the message will be considered invalid and be rejected. This ensures the "freshness" of the message.

Note: The message expiration time should be set properly to accommodate the network latency for the messages to transfer from source to destination and the clock skewness between the clocks of the sender and the receiver.

Advanced Options

The following advanced options can be configured:

Actor

This parameter specifies a URI that should be set as the actor attribute of WS-Security header associated with the timestamp header to be verified. Example syntax could look like this:

  mustUnderstand="…">
...
</wsse:Security>

The optional actor attribute can be used to indicate which recipient should process the timestamp header. If the actor is omitted, it means that timestamp header is targeted to final endpoint. The Actor value must conform to a supported Uniform Resource Identifier (URI) format.

Note: This implemention of the "WS-Security Insert Timestamp Component" supports "mustUnderstand=True" by default.

Generate Audit Data Audit

If this parameter is checked, the execution status of this component will be logged as part of audit trail. If this option is not selected, the execution status of this component will be logged only when there is error while executing this component.
