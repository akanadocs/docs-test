Introduction


The Detailed Auditing Policy is an out-of-box policy that is that is part of the Policy Manager default installation. It is instance the Auditing Service Policy and provides detailed auditing of messages and records the message metrics in the Monitoring > Logs tab as well as the messages of each exchange. The policy is located in the Policies folder of the Root Organization.

The following message metics are recorded.

  • Input Message - Audits Input Message content.
  • Output Message - Audits Output Message content.
  • Fault Message - Audits Fault Message content.
  • Message Size - Audits of the size of messages in the exchange.
  • Binding - Audits binding information.
  • Transport - Audits Transport information.
  • Contract - Audits contract governing a message.

The following identities are audited:

  • Consumer - Audits consumer identity sent with a request message.
  • End User - Audits end-user identity sent with a request message.

Configuration

Let's take a quick walkthrough of the Detailed Auditing Policy configuration process to get your started.

Step 1: Attach Policy

Go to the Policies folder in the Root Organization and attach it to a web service, operation, or binding that you would like to audit.

Step 2: Test Policy and View Monitoring Data

After you've attached the Detailed Auditing Policy to a service, operation, or binding, send a request to your service and go to the Services > Monitoring section to view the results for Logs, Real Time Charts, and Historical Charts. Refer to the Policy Manager Online Help (i.e., Help button) for more information on the using the monitoring functions.
