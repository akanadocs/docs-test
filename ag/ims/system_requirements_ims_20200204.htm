System Requirements for Intermediary for Microsoft with Akana Platform 2020.2.4

The Intermediary for Microsoft product relies on a compatible version of the Akana Platform, including compatible versions of all applicable resources such as database types and versions, hardware, and operating system software.

For details of the system requirements for Akana Platform 2020.2.4, see For supported versions, see: System Requirements for Akana Platform 2020.2.x.

This document provides supplementary information on recommended versions that Intermediary for Microsoft was tested with for Akana Platform 2020.2.4. In most cases, the line items below are a subset of those supported for the Akana Platform. For example, specific database versions are listed below, so it's important to use one of those. Some information, such as Microsoft IIS version, is additional information not applicable to the default system requirements.

Prerequisites

To install the Intermediary for Microsoft product, you must first have the following software products installed:

  • Akana Policy Manager
  • A relational database
  • A Windows Server machine

Database versions

{we don't need to list this unless there is a subset for IMS. Probably there is.}

Akana API Platform 2020.2.x supports the following database versions:

  • MySQL 8.0

    Database driver required: see Database drivers (install doc).

  • Oracle 12c Release 2, Oracle 18c (18.3), Oracle 19c.

    Note: The Oracle database password must not start with a number or special character.

    Database driver required: see Database drivers (install doc).

  • Microsoft SQL Server:
    • 2012 SP4

      Note: Akana support of Microsoft SQL Server 2012 SP4 ends in July 2022.

    • 2016 SP2
  • MongoDB (Windows 64-bit MSI and Linux 64-bit TGZ):
    • 4.2.11 or later 4.2.x versions

    To download a MongoDB file for your platform, refer to the MongoDB Downloads website.

Note: Depending on the database type and version that you're using, there might be specific requirements, constraints, or additional steps needed. For details, refer to the Database Notes section of the installation documentation.
