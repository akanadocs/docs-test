Script Management Functions

The following script management functions apply to scripts defined in the Scripts folder. They are available on the Script Summary, the Script Summary drop-down menu, or via the Actions Portlet on the Script > Details page.

Add Script

Allows you to define common functions that you may need to perform in your scripts and save them in Script objects.

  • To make use of those common functions from a Script Activity or QoS Script Policy, the Script objects defining those functions must be imported.
  • The Imports area is where those scripts are identified for import.
  • Any number of Scripts in the tree can be moved to the Imported Scripts list box.
  • Only scripts that match the language type will be available for selection in the tree.
  • The script does not return a value.

Modify Script Information

Allows you to change the name and description of a script.

Change Organization

Allows you to move the current script configuration to a new "Organization."

  • Note that script referenced in process definitions via the Script Activity, can only be moved within the current Script Scope (i.e., Organization, Service, Operation).
  • If you wish to move a script to a different "Organization" the references must stay in scope.
  • To determine what references a script might have, select "View Script References" for the current script.
  • To remove script references, remove the Script Activity that contains the reference from the script definition.

back to top

Copy Script

Allows you to replicate a script definition and assign a new "Script Name" and "Script Key." Using this function you can configure elements that represent core functionality of a script, replicate them to a new script, and then perform additional customization on the copy.

back to top

Delete Script

Deletes the selected script definition. In order to delete a script, it cannot be "Referenced" (i.e., used/imported in one or more Script Activity definitions). The number of script references are indicated in the "#" column and can be viewed by selecting the "View Script References."

If you would like to delete a script, you must remove the script from the list of imported scripts in the Script Activity in your process definition.

back to top

Export Script

Provides a method of exporting a script definition to a "Package" file. This "Package" file can then be imported into a different Policy Manager deployment using the "Import Package" function.
