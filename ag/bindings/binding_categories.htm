Binding Categories

Learn how to view current binding category schemes and how to assign/unassign categories for the current binding definition.

View Binding Categories Summary

See a listing of categories that are assigned to the current binding.

To view a binding categories summary

  1. Go to Configure > Registry > Bindings.
  2. On the Bindings Summary screen, click View Binding Details.
  3. On the Bindings Details screen, click the Categories tab. The Binding Categories Summary screen is displayed.

Assign Binding Categories

Navigate the Category Hierarchy to select a Category Scheme, navigate the tiers of the Category Scheme to select the Category Values, and apply the selected values to the current binding definition.

To assign binding categories

  1. Go to Configure > Registry > Bindings.
  2. On the Bindings Summary page, choose the binding you want to assign a binding category to.
  3. From the Actions drop-down list, select View Binding Details.
  4. On the Binding Details screen, click the Categories tab. The Binding Categories Summary screen is displayed, showing all the categories that are defined for the current binding.
  5. Click Assign Category. The Select Categories screen is displayed, showing the Category Hierarchy.
  6. Select a Category Scheme.
  7. Navigate the tiers of the Category Scheme and click on a Category Value to apply to the current binding definition.
  8. Click Finish. The system assigns the selected category value to the current organization definition, and the Add Categories Summary screen displays.
  9. Click Close.

Unassign Binding Categories

Remove a category association from the current binding definition. Necessary if:

  • The established association with a selected category scheme is no longer valid.
  • A binding definition has changed.

To unassign binding categories

  1. Log in to the Policy Manager Management Console.
  2. Go to Configure > Registry > Bindings.
  3. On the Bindings Summary screen, click on the binding you want to assign a binding category to.
  4. From the Actions drop-down list, select View Binding Details.
  5. On the Binding Details screen, click the Categories tab. The Binding Categories Summary screen is displayed, showing all the categories that are defined for the current binding.
  6. Select the category scheme to be unassigned.
  7. Click Unassign Category.
  8. Click OK.

Assign Binding Category Group

This section describes how to select a UDDI Category Group that you want to assign new categories to, navigate the Category Hierarchy to select a Category Scheme, navigate the tiers of the Category Scheme to select the Category Values, and then apply the selected values to the binding definition. When you've applied the category values, a category definition is created and the record is displayed on the Category Summary screen.

Binding Category Groups allow you to categorize service relationships with a higher level of granularity, and contain a list of business categories that describe specific business aspects of the Organizations, Services, tModels, Schemas, Interfaces, or Bindings.

Examples of Category Groups are:

  • Industry
  • Product category
  • Geographic region

Note: Each Organization, Service, tModel, Schema, Interface, or Binding can be assigned to one or more Category Groups.

To assign a binding category group

  1. Log in to the Policy Manager Management Console.
  2. Go to Configure > Registry > Bindings.
  3. On the Bindings Summary screen, click on the binding you want to assign a binding category to.
  4. From the Actions drop-down list, select View Binding Details.
  5. On the Binding Details screen, click the Categories tab. The Binding Categories Summary screen is displayed, showing all the Categories that are defined for the current Binding.
  6. Click Assign Category Group. The Assign Category Group Wizard launches and the Select Category Group page is displayed, showing a list of Category Groups that exist for the current binding.
  7. Select the radio button next to the Category Group that you want to assign new categories to.
  8. Click Next. The Select Categories screen displays.
  9. Navigate the tiers of the Category Scheme and select a Category Value to apply to the current binding definition.
  10. Click Finish, and then click Close.

Unassign Binding Category Group

Remove a category group association from the current binding. This is necessary if:

  • The established association with a selected category group is no longer valid.
  • A binding definition has changed, requiring a new category association.

To unassign a binding category group

  1. Log in to the Policy Manager Management Console.
  2. Go to Configure > Registry > Bindings.
  3. On the Bindings Summary screen, click on the binding you want to assign a binding category to.
  4. From the Actions drop-down list, select View Binding Details.
  5. On the Binding Details screen, click the Categories tab. The Binding Categories Summary screen is displayed, showing all the categories that are defined for the current binding.
  6. From the line item display, select the category group to be unassigned.
  7. Click Unassign Category Group.
  8. Click OK.
