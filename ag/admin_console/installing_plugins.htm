Installing SOA Software Platform Plug-ins

Learn how to install plug-ins that are part of the SOA Software Platform default installation.

SOA Software Cluster Support

This plug-in provides support for group administration of multiple containers via a single SOA Software Administration Console. The clustering feature is based on the idea of a Master node and a set of Slave nodes that all share a common set of feature bundles and configuration. When the clustering feature is installed, any Container Instance may be configured as either a Master or Slave. All Slaves will periodically synchronize their state with the Master node. If the Master node becomes unavailable for any reason, the slaves will remain in their last configured state.

SOA Software External Keystore Feature

This plug-in allows you to enable a Hardware Security Module (HSM) security provider as your cryptography solution for storing and managing Policy Manager PKI keys and certificates. After the feature is installed, all key management tasks performed using the Manage PKI Keys Wizard for each identity (service, organization, container, or user) will be stored in the designated external keystore.

SOA Software Kerberos Impersonation

This plug-in enables kerberos constrained delegation, or impersonation. Requires Java 1.8.
