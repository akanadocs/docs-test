PUT /api/businesses/{BusinessID}/connectionsettings

Updates the configurable connection settings for a business on the platform.

Authorization Roles/Permissions: Must be an authorized administrator: Business Admin or Site Admin

PUT

URL

https://<hostname>/api/businesses/{BusinessID}/connectionsettings

Back to top

Sample Request

The example below shows a request to update the settings for sandbox and production environments. In the sandbox environment, the comments workflow is reset to the system default.

Request URL

https://<hostname>/api/businesses/tenantbusiness.acmepaymentscorp/connectionsettings

Sample request headers

Content-Type: application/json
Accept: application/vnd.soa.v72+json
X-Csrf-Token_acmepaymentscorp: <TokenID>

Sample request body

{
  "WorkflowSetting" : [ {
    "WorkflowDefinitionKey" : "workflow:definition:apicontract:v72",
    "Environment" : "Production",
    "ConnectionCommentWorkflowDefKey" : "workflow:definition:comment"
  }, {
    "WorkflowDefinitionKey" : "workflow:definition:apicontract:v72",
    "Environment" : "Sandbox",
    "ConnectionCommentWorkflowDefKey" : ""
  } ]
}

Request Parameters

Parameter Parm Type Data Type Required Description
BusinessID Path string Required The unique ID for a specific business on the platform.
ConnectionSettings Body ConnectionSettings Required Contains information about a business's settings in relation to connections (app/API contracts).

Response

If successful, this operation returns HTTP status code 200, with the updated settings.
