GET /api/businesses/{BusinessID}/resources

Returns information about business resources (scopes).

Authorization Roles/Permissions: Must have read permission for the business information.

Response Body

The response body is in the form of an RSS channel, and includes the items listed below. The RSS version is 1.0. The title of the RSS channel is Business Resources. There is also a description value indicating the BusinessID for the business. Each item in the channel represents one business scope, and includes the information listed below.

Name Description
title The title of the business resource.
description The text description for the business resource. For scopes, it is the "short description" specified for the scope.
category[ ]

An array of sets of two name/value pairs, each including Value and Domain, with a value for each. Categories include:

  • uddi:soa.com:resourcetype: value is always resource.
  • uddi:soa.com:visibility: indicates the visibility setting for the resource.
  • uddi:soa.com.resource:default-oauth-grant: indicates whether "OAuth default scope" is enabled for the resource.
  • uddi:soa.com.resource:grant-user-auth-required: indicates whether "user authorization required" is specified for the resource.
  • uddi:soa.com.access:production-access-anonymous-allowed: indicates whether anonymous access is allowed for production access.
  • uddi:soa.com.access:sandbox-access-anonymous-allowed: indicates whether anonymous access is allowed for sandbox access.
comments The comment information entered for the resource. For scopes, it is the "long description" specified for the scope..
GUID GUID for the resource. It is the unique ResourceID; for example, fb73518a-2318-4088-bd27-faa442743ef1.acmepayments.
