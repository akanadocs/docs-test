POST /content/upload

Uploads a single file of existing content to the platform, or uploads a ZIP file of existing content and unzips it to the specified location.

Note: When creating or uploading content files, you can use the POST /content/upload operation to upload a single file, or multiple files in a ZIP, using the multipart/form-data content type. You can use the POST /content/{path} operation to create a new folder with a default JSON table of contents file in it, as in the example below, or to create a new file. If you are creating a new file, it cannot be empty; you must include the file contents in the payload, and must use the application/x-www-form-urlencoded content type.

Authorization Roles/Permissions: API Admin or other authorized user.

Sample Request

The example below shows a request to upload a ZIP file of content for the specified API.

Request URL

https://api.acmepaymentscorp.com/content/api/api19427.acmepaymentscorp/documents?unpack=true&wrapInHTML=true

Sample request headers

POST /content/api/api19427.acmepaymentscorp/documents?unpack=true&wrapInHTML=true HTTP/1.1
Host: <hostname>
Accept: text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8

Sample request body

The request body is the ZIP file.

Request Parameters

Parameter Parm Type Data Type Required Description
path Path string Required The path to the location where the content being uploaded will be unzipped and stored. The path must be valid for your API resources.
unpack Query boolean Required Indicates whether the content must be unzipped. For a ZIP file this parameter is requied and must be set to true.
InMultiPart Body InMultiPart Required The content being uploaded.

Note: For information about the wrapInHTML parameter shown in the sample request message above, see File Upload with Ajax. This is only used if you are running the API in a browser context.
