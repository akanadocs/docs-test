PUT /api/apis/versions/{APIVersionID}

Updates information about an API version.

Authorization Roles/Permissions: Must have permission to modify the API; an API Admin or Business Admin.

This topic includes the following sections:

  • HTTP Method
  • URL
  • Sample Request
  • Request Headers
  • Request Parameters
  • Response
  • Sample Response
  • Response Headers
  • Response Body
  • Error Codes/Messages

HTTP Method

PUT

Back to top

URL

https://<hostname>/api/apis/versions/{APIVersionID}

Back to top

Request Headers

For general information on request header values, refer to HTTP Request Headers.

Header Description
Accept application/json or application/xml
Content-Type application/json or application/xml
X-Csrf-Token_{fedmemberID} The CSRF prevention header; may or may not be required, depending on platform settings. See CSRF Prevention on the Platform.

Back to top

Request Parameters

Parameter Parm Type Data Type Required Description
APIVersionID Path String Required The unique ID for a specific API version.
APIVersion Body APIVersion Required Includes information about an API version. The entire version, including the changed data.

Response

If successful, this operation returns HTTP status code 200, with the updated API version object.
