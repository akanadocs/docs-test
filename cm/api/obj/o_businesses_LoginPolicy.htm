LoginPolicy Object

Contains details about login policy settings for a business on the platform

This object is used by the following operations:

  • PUT /api/businesses/{BusinessID}/loginpolicy
  • GET /api/businesses/{BusinessID}/loginpolicy

The properties included in the LoginPolicy object are listed below.

Note: The feature covering the first three options is disabled by default.

Property Type Description
MaxSequentialFailureAttemptsAllowed integer Indicates the maximum number of consecutive failed login attempts before the user's account is disabled.
AttemptsPeriod integer Indicates the period of time, in minutes, over which the number of failed login attempts is calculated.
SuspensionMinutes integer Indicates the period of time, in minutes, for which the user's account is locked after failed login attempts.
AutoLoginExternalDomain string

If this setting is enabled, the platform attempts to log in via an external domain (LDAP domain) when using login based on email address and password. This setting is present only for backwards compatibility

Default: com.soa.disabled
LoginSessionTimeout integer

Deprecated: supported for backward compatibility only. Replaced by ActiveSessionTimeout and InactiveSessionTimeout.

Indicates the period of time, in minutes, after which the user is automatically logged out of the Default Theme or Simple Dev developer portal user interface if the session is inactive. Default: 30 minutes.
ActiveSessionTimeout integer

Indicates the period of time, in minutes, after which the user is automatically logged out of the Default Theme or Simple Dev developer portal user interface even if the session is active. Default: 30 minutes.

Note: ActiveSessionTimeout is the only timeout setting that is applicable to API users who are not using the developer portal user interface.
InactiveSessionTimeout integer

Indicates the period of time, in minutes, after which the user is automatically logged out of the Default Theme or Simple Dev developer portal user interface if the session is inactive. Default: 30 minutes.

Note: the InactiveSessionTimeout setting takes precedence over LoginSessionTimeout which will be deprecated.
PersistentSession Boolean Indicates whether the session cookie persists when the browser is closed. Default; true. If set to false, the cookie expires when the browser is closed.
