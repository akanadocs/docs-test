EmbeddedNodeConfiguration Object

Contains information about the Elasticsearch embedded node configuration for a container.

This object is used by the following operations:

  • GET /api/index/{ContainerKey}/nodeconfig (in the response)
  • PUT /api/index/{ContainerKey}/nodeconfig (in request and response)
  • Operations that reference the EmbeddedNodeConfigurations object

The properties included in the EmbeddedNodeConfiguration object are listed below.

Property Type Description
ContainerKey string The unique key for the container that this configuration is for.
Name string The name of the container.
IndexLocation string The location of the Elasticsearch index.
BindToAll boolean Indicates bind to all interfaces.
NetworkBindAddress string The specific network bind address. If Bind to all is selected, the Elasticsearch feature binds itself to the value 0.0.0.0, which indicates all addresses of the machine; in this case, the network bind address is not necessary/used.
NetworkPublishAddress string The address that other nodes will use to communicate with this node. It can be the machine host name or a valid IP address for the machine.
TransportPort integer The port used for node-to-node communication, if there are multiple nodes. Default: 9300.
HttpPort integer The port used for the REST API. Default: 9200.
HttpEnabled boolean

Indicates that the Elasticsearch node is HTTP-enabled.

There are two ways that the data node or master node receives the Elasticsearch search request; TCP or HTTP. TCP is always enabled for all nodes, but HTTP is a higher level protocol and is optional. To enable it, check the box.

The product does not depend on the HTTP port of any container. The HTTP enabled option really only comes into play to facilitate using some other interface, from outside the product, with HTTP interfaces. For example, you might want to use the HTTP interface of the Elasticsearch node for a third-party client, or for troubleshooting purposes. Internally, the platform does not depend on the HTTP port.
MasterEligible boolean Indicates whether this is a master eligible node. There must be at least one master eligible node in a cluster.
DataNode boolean

Indicates whether this is a data node. The data node is where the index is stored. There must be at least one data node in a cluster.

Index location is required only for data nodes. It's vital because if the node doesn't hold the data, there is no storage of index location. Index location is where the index is saved, and the index is only saved on the data nodes. Therefore, for a data node, the index location needs to be provided.

The default index location is a subfolder to the installation folder, /index, at the same level in the folder structure as the /lib and /bin folders. Index data is stored in this folder.

If the node is not a data node, index location is not needed.
