ApplicationVersion Object

Contains information about a version of an app.

For example, this object is returned by the GET /api/apps/versions/{RuntimeID} method.

This object is used by the following operations:

  • POST apps/{AppID}/versions
  • PUT apps/versions/{RuntimeID}
  • GET /api/apps/versions/{RuntimeID}

The properties included in the ApplicationVersion object are listed below.

Property Type Description
RuntimeID string Unique ID used at runtime by the app version.
AppID string The unique ID for a specific app.Required when adding or modifying an app version.
Name string The display name of the app version; for example, 1.0. Required when adding or modifying an app version.
Description string Text description of the app. Required when adding or modifying an app version.
WebsiteAddress string The website for the app, if defined.
Visibility string The visibility setting for this resource. For a list of valid values, see Visibility Settings.

Required when adding an app version.
InitialEnvironment string

The initial environment for the app (normally Sandbox).

Required when adding or modifying an app version.
CurrentEnvironment string The current environment for the app (Sandbox or Production).
Tag string[ ] One or more user-defined tags for the app version.
HasKeyInfo boolean Indicates whether or not the app has a security key certificate loaded (true or false).
Created dateTime The date/time stamp when the resource was first created. Used only in response messages.
Updated dateTime The date/time stamp when the resource was last updated. Used only in response messages.
State string The current state of the app. For valid values, see App State Values.
SecurityMechanisms string[ ] One or more security mechanisms being used by the app version. For more information, see App Security Mechanisms.
RatingSummary RatingSummary An aggregated, averaged summary of individual user ratings for the resource, including the current ratings count for each possible value (1, 2, 3, 4, or 5). Used only in response messages.
Identity string When adding an app version, you can specify a custom identity name for the app version. If no custom value is specified, in response messages this property defaults to the reverse of the RuntimeID, with a dash. For example, if the RuntimeID is 123456.acmepayments, the value for the Identity property would be open-123456.
SharedSecret string The Shared Secret value for the app, if applicable.
EncryptedSharedSecret string The encrypted Shared Secret value for the app, if applicable.
AppVersionSettings AppVersionSettings Contains information about the settings for an app version.
TargetResources TargetResources Contains information about one or more resources that are associated with the current resource.
