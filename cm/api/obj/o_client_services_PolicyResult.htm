PolicyResult Object

Provides information about the instructions to be implemented as a result of application of the AtmosphereApplicationSecurityPolicy; for example, how the token should be added to the request message (header, cookie, query parameter, or POST parameter) and what steps should be taken in the event of an error.

The properties included in the PolicyResult object are listed below.

Property Type Description
RequestContext RequestContext Includes information about the request context—that is, the instructions about how the request message should be prepared for sending to the API. Uses the information returned by one of these operations: POST /api/client/services/request/client/identity (assign client identity for shared secret, POST) PUT /api/client/services/request/client/identity (assign client identity for shared secret, PUT). Uses the response to one of these operations to prepare the request message in conformance with the settings of the AtmosphereApplicationSecurityPolicy.
Proceed boolean Indicates whether there is a problem with the message: true = no problem, proceed with the message.
PolicyStatus PolicyStatus

If there is an issue, indicates the status, with the following information:

  • Status: string indicating the status.
  • ErrorCode: code for the error.
  • ErrorMessage: Message associated with the error.
