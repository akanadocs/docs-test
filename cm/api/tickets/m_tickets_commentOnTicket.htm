POST /api/tickets/{TicketID}/comments

Adds a comment to the specified ticket.

Authorization Roles/Permissions: The user must be logged in, and must be an authorized user for the resource the ticket relates to.

Sample Request

The example below shows a comment added to the specified ticket.

Request URL

https://<hostname>/api/tickets/ticket25687.acmepaymentscorp/comments

Sample request headers

POST /api/tickets/ticket25687.acmepaymentscorp/comments HTTP/1.1
Host: <hostname>
Accept: text/plain, */*; q=0.01
Content-Type: application/json; charset=UTF-8
X-Csrf-Token_{tenant}: <TokenID>

Sample request body

{
  "Content":"FYI I've also experienced the same problem. Really glad you are getting it fixed.",
  "UserID":"b302bcd5-23cb-4e13-9135-68a7a88e729e.acmepaymentscorp"
}

Request Parameters

Parameter Parm Type Data Type Required Description
TicketID Path string Required The unique ID for a specific ticket.
Comment Body Comment Required Contains information about a comment on a Board item such as a ticket or discussion.

Response

If the call is unsuccessful an error code/message is returned. One or more examples of possible errors for this operation are shown below.

Item Value
