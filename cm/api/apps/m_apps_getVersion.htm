GET /api/apps/versions/{RuntimeID}

Returns a version of an app.

Authorization Roles/Permissions: App team member, Business Admin, or Site Admin.

This topic includes the following sections:

  • HTTP Method
  • URL
  • Sample Request

HTTP Method

GET

Back to top

URL

https://<hostname>/api/apps/versions/{RuntimeID}

Back to top

Sample Request

The example below requests the version information for the ID in the URL.

Sample request URL #1

https://<hostname>/api/apps/versions/4iXro2Uxs2h4LviCshR9pK9w.acmepayments

Sample request URL #2

https://<hostname>/api/apps/versions/4iXro2Uxs2h4LviCshR9pK9w.acmepayments?IncludeSettings=true

Sample request headers

Accept: application/json, text/javascript, */*; q=0.01
Host:<hostname>
X-Csrf-Token_<tenant>: <TokenID>

Sample request body

Not applicable.

Request Parameters

Parameter Parm Type Data Type Required Description
RuntimeID Path string Required Unique ID used at runtime by the app version.
IncludeSettings Query boolean Optional Indicates whether the settings for the resource should be included in the information returned. If not specified, defaults to false.

Back to top

Response

If successful, this operation returns HTTP status code 200, with the version information in a JSON or XML object.

Error Codes/Messages

If the call is unsuccessful an error code/message is returned. One or more examples of possible errors for this operation are shown below.

Item Value
204 No content. The server successfully processed the request, but is not returning any content. For example, you might get this error if you specified an invalid RuntimeID.
401 Unauthorized. For example, you would get this response if you didn't include the custom X-Csrf-Token_fedmemberID header in the request, or if it was invalid or expired.
404 The resource could not be found.
500 An error occurred processing the call.
