GET /api/apps/versions/{AppVersionID}/contracts

Returns information about API contracts for the specified app version, in the form of a JSON object.

Authorization Roles/Permissions: Must have read permission for the resource.

This topic includes the following sections:

  • HTTP Method
  • URL
  • Sample Request
  • Request Headers
  • Request Parameters
  • Response
  • Sample Response
  • Response Headers
  • Response Body
  • Error Codes/Messages

HTTP Method

GET

Back to top

URL

https://<hostname>/api/apps/versions/{AppVersionID}/contracts[?Environment=environment]

Back to top

Sample Request

The example below shows a request for contracts for the specified app version.

Request URL #1

Returns results for all environments.

https://<hostname>/api/apps/versions/2mWn90b8Y4C2vL7qCxDW8f38.acmepayments/contracts

Request URL #2

With optional parameter, returns results for Sandbox only.

https://<hostname>/api/apps/versions/2mWn90b8Y4C2vL7qCxDW8f38.acmepayments/contracts?Environment=sandbox

Sample request headers

Accept: application/vnd.soa.v71+json
X-Csrf-Token_<tenant>: <TokenID>

Sample request body

Not applicable.

Back to top

Request Headers

For general information on request header values, refer to HTTP Request Headers.

Header Description
Accept application/vnd.soa.v71+json, application/vnd.soa.v72+json

Back to top

Request Parameters

Parameter Parm Type Data Type Required Description
AppVersionID Path string Required The unique ID for a specific app version.
Environment Query string Optional The environment (Sandbox or Production). If not specified, results are returned for all environments.

Back to top

Response

If successful, this operation returns HTTP status code 200, with information about the contracts applicable to the specified app version.
