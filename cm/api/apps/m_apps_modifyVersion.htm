PUT apps/versions/{RuntimeID}

Changes one or more values associated with an app version.

Authorization Roles/Permissions: App team member, Business Administrator

Authorization token renewal: This operation changes information that is reflected in the authorization token; therefore, when invoking this operation, you must also renew the token.

Sample Request

The example below modifies the name (version number) and tags of an app. The operation sends the new app version information in JSON format.

Sample request URL

https://<hostname>/api/apps/versions/4asLaEWAR0S4eV9aMRJcGK9V.acmepayments

Sample request headers

Accept: text/plain
Content-Type: application/json; charset=UTF-8
Host:<hostname>
X-Csrf-Token_<tenant>: <TokenID>

Sample request body

{
  "Name":"1.01",
  "Description":"Card game",
  "Tag":[
    "cards",
    "game",
    "group",
    "updated"
  ],
  "AppID":"app10017.acmepayments",
  "InitialEnvironment":"Sandbox"
}

Response

If successful, this operation returns HTTP status code 200, with the RuntimeID of the app version in plain text.
