PUT /api/client/services/request/client/identity

Assigns the client identity, using a shared secret value. Uses HTTP PUT. Generally used when updating an existing shared secret value.

This operation works essentially the same as the corresponding POST operation. For information about using it, refer to POST /api/client/services/request/client/identity
