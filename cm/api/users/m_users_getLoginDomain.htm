GET /api/users/{UserID}/logindomain

Returns information about the login domain for the specified user.

Authorization Roles/Permissions: Site Admin.

HTTP Method

GET

Back to top

URL

https://<hostname>/api/users/{UserID}/logindomain

Sample Request

The example below shows a request for the login domain for the specified user.

Request URL

https://<hostname>/api/users/user10002.acmepaymentscorp/logindomain

Sample request headers

Accept: application/json
X-Csrf-Token_{tenant}: <TokenID>

Sample request body

Not applicable.

Request Parameters

Parameter Parm Type Data Type Required Description
UserID Path string Required The unique ID for a specific user.
cache Query boolean Optional Indicates whether the information should be cached. If you don't want the domain information to be read from cache, specify false. If not specified, defaults to true. For more information about caching on the platform, see How the Enterprise API Platform API Uses Caching.

Back to top

Response

If successful, this operation returns HTTP status code 200, with information about the login domain for the user specified in the request message.

If the call is unsuccessful an error code/message is returned. One or more examples of possible errors for this operation are shown below.

Item Value
