Example: Log in with Google

This example outlines the steps so that a user can log in using a Google account. Some of these steps require Site Admin setup, and the login steps can be done by anyone with a Google account.

Some external setup is needed, because there must be an account with Google, with credentials, so that the developer portal itself will be recognized and trusted with the user's ID token.

The Site Admin must:

  1. Set up an account with Google.
  2. Safely store the App ID and App Shared Secret needed to identify the platform.

Then, the Site Admin must:

  1. Log in via the API, POST /api/login operation
  2. Add a Google domain.

    Note: The Site Admin can also create the domain via the developer portal user interface: Admin > Domains, click Add Domain button and choose OpenID Connect Relying Party. When the Site Admin enters the well-known configuration URL, the platform user interface reads all the values from the URL.

  3. Enable the Google domain as a login domain. In the developer portal user interface: More > Admin > Logins.

At this point, the domain is ready for the user to log in and add an app:

  1. The user logs in with the Google credentials.
  2. For first-time login only, the user must provide additional profile details.
  3. The user can now perform any general functions, such as adding an app, either in the user interface or by using the POST /api/apps operation.

Note: to perform certain actions such as adding an API, the user must have an additional role. To add the user as an API Admin, perform these additional steps:

  1. Site Admin: Invites the user to the API Admin role.
  2. User: accepts the invitation.
  3. The user can now add an API, either in the user interface or by using the POST /api/apis operation.

When done, each user should log out using the GET /api/login/endsession operation. Use the following values:

  • URL: https://{hostname}/api/login/endsession
  • Accept header: application/json
