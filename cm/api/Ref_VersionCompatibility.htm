Version Compatibility Notes

This document outlines changes in specific operations, between versions. When upgrading from one version to another, you might need to adjust your implementation to accommodate changes in the operations listed below.

Note: This documentation set includes instructions for the latest version only.

7.2 to 7.2.1

Apps Service

PUT apps/versions/{AppVersionID}/secret:

  • In version 7.2, this operation takes any content-type value and no payload.
  • In version 7.2.1, it requires a Content-Type header of application/x-www-form-urlencoded and a comment in the payload.
Users Service

POST /api/users (Site Admin only):

  • In version 7.2, this operation has a response content-type of text/plain and returns the UserID of the new user.
  • In version 7.2.1 it returns the User object, with a response Content-Type (and, therefore, request Accept header value) of application/json, application/vnd.soa.v71+json, or application/vnd.soa.v72+json.
Apps Service

Deprecated:

PUT apps/versions/{AppVersionID}/secret

Replaced by:

POST /api/apps/versions/{RuntimeID}/keyinfo

Note apps/addKeyInfo is deprecated in this release, replaced by apps/saveKeyInfo which has the same endpoint.
