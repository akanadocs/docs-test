POST /oauth/auz/grants

Authorizes the grant. The resource owner authorizes the client app based on the OAuth scopes that have been configured. As a result of this operation, the grant status changes from Pending to Active.

In the developer console, this operation is used by Test Client in the OAuth authorization page. When the user clicks the Authorize button, this operation is called. By clicking the button, the user is authorizing the client to invoke APIs on his/her behalf with the specified scopes.

Authorization Roles/Permissions: Must be the resource owner.
